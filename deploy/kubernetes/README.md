# Deploying Sock Shop on Kubernetes

All services are built from source and deployed with SHA-based image tags
(e.g. `dev-a3f7c21`). From the repository root:

## Build and deploy

```bash
./bin/build --kind && ./bin/deploy --kind    # kind (local)
./bin/build --eks && ./bin/deploy --eks      # EKS
./bin/build --gke && ./bin/deploy --gke      # GKE
```

See [DEVELOP.md](../../DEVELOP.md) for full details.

## Kustomize structure

```
manifests/              Base resources (all services)
overlays/
  kind/                 Generated by bin/deploy — overrides image tags with SHA-based tags
  eks/                  Generated by bin/deploy — overrides to ECR URIs with SHA-based tags
  gke/                  Generated by bin/deploy — overrides to Artifact Registry URIs with SHA-based tags
```

Both overlays reference `manifests/` as the base. The base manifests use
upstream image tags as Kustomize defaults; they are never deployed directly —
all deployments go through an overlay generated by `bin/deploy`.

## Carts and MongoDB version

The carts service uses **Spring Boot 2.7**, **Java 17**, and the **MongoDB Java
driver 4.x** (via Spring Data MongoDB 3.x). It talks to MongoDB over the modern
wire protocol, so carts-db in the manifests uses **mongo:7+** (driver 4.x
supports MongoDB 5–8).

## Monitoring

All monitoring is performed by Prometheus. All services expose a `/metrics`
endpoint with a `request_duration_seconds` histogram.

The manifests for monitoring are in [manifests-monitoring](./manifests-monitoring)
and [manifests-alerting](./manifests-alerting/).
