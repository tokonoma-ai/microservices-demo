### Create command

```bash
eksctl create cluster \
  --name tokonoma \
  --region us-west-2 \
  --version 1.31 \
  --nodegroup-name workers \
  --node-type m7i-flex.large \
  --nodes 3 \
  --nodes-min 2 \
  --nodes-max 4 \
  --managed \
  --with-oidc
```

**Note:** `t3.medium` was rejected by an SCP restricting the account to Free Tier
eligible types. `m7i-flex.large` works.

### Post-cluster setup

```bash
# EBS CSI driver addon
aws eks create-addon \
  --cluster-name tokonoma \
  --addon-name aws-ebs-csi-driver \
  --region us-west-2

# Attach EBS policy to node role
# Node role name: eksctl-tokonoma-nodegroup-workers-NodeInstanceRole-TzoQGV5hX5wg
aws iam attach-role-policy \
  --role-name eksctl-tokonoma-nodegroup-workers-NodeInstanceRole-TzoQGV5hX5wg \
  --policy-arn arn:aws:iam::aws:policy/service-role/AmazonEBSCSIDriverPolicy

# Update kubeconfig
aws eks update-kubeconfig --name tokonoma --region us-west-2
```

**Important:** The node role name is generated by eksctl and will differ on each
cluster creation. The script must discover it dynamically:

```bash
NODE_ROLE=$(aws eks describe-nodegroup \
  --cluster-name tokonoma \
  --nodegroup-name workers \
  --region us-west-2 \
  --query 'nodegroup.nodeRole' \
  --output text)
ROLE_NAME=$(basename "$NODE_ROLE")
```

### S3 bucket setup

Create the S3 bucket that Quickwit will use for index storage (replaces MinIO):

```bash

  --region "$REGION" \

```

### IAM policy for Quickwit S3 access

Create an IAM policy scoped to the Quickwit bucket:

```bash
cat > /tmp/quickwit-s3-policy.json <<'EOF'
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject",
        "s3:DeleteObject",
        "s3:ListBucket",
        "s3:GetBucketLocation"
      ],
      "Resource": [
        "arn:aws:s3:::tokonoma-quickwit-indexes",
        "arn:aws:s3:::tokonoma-quickwit-indexes/*"
      ]
    }
  ]
}
EOF

aws iam create-policy \
  --policy-name QuickwitS3Access \
  --policy-document file:///tmp/quickwit-s3-policy.json
```

Policy ARN: `arn:aws:iam::853962316687:policy/QuickwitS3Access`

### IRSA (IAM Roles for Service Accounts) for Quickwit

IRSA lets Quickwit pods assume an IAM role to access S3 without static
credentials. The `--with-oidc` flag in the cluster creation command already
set up the OIDC provider.

```bash
eksctl create iamserviceaccount \
  --cluster tokonoma \
  --region us-west-2 \
  --namespace qw \
  --name quickwit-sa \
  --attach-policy-arn arn:aws:iam::853962316687:policy/QuickwitS3Access \
  --approve \
  --override-existing-serviceaccounts
```

This creates a Kubernetes ServiceAccount `quickwit-sa` in namespace `qw`
annotated with the IAM role ARN. Quickwit pods using this service account
will automatically receive temporary AWS credentials via the OIDC token.

**Verify:** After creation, confirm the annotation:

```bash
kubectl get sa quickwit-sa -n qw -o yaml
# Should show: eks.amazonaws.com/role-arn: arn:aws:iam::853962316687:role/...
```

## 5. Deploy: Platform Stack (namespace `qw`)

The existing `2o/platform/bin/deploy` script works for kind (with MinIO). For
EKS with S3, the key differences are:

1. **StorageClass** — EKS uses `gp2` (not `standard`)
2. **Image source** — ECR URIs instead of `kind load`
3. **Storage backend** — S3 bucket (no MinIO)
4. **Credentials** — IRSA (no static access keys)

### Quickwit values for EKS (`values-eks.yaml`)

Create `2o/platform/config/quickwit/values-eks.yaml` — derived from
`values-local.yaml` but pointing to real S3 instead of MinIO:

```yaml
# Quickwit Helm values for EKS (S3 backend via IRSA)

searcher:
  replicaCount: 1

indexer:
  replicaCount: 1

metastore:
  replicaCount: 1

janitor:
  enabled: true

control_plane:
  enabled: true

# Use the IRSA service account for S3 access
serviceAccount:
  create: false
  name: quickwit-sa

environment:
  NO_COLOR: "1"
  QW_METASTORE_URI: s3://tokonoma-quickwit-indexes/

config:
  default_index_root_uri: s3://tokonoma-quickwit-indexes/

  indexer:
    enable_otlp_endpoint: true

  storage:
    s3:
      region: us-west-2
      # No endpoint, flavor, or static credentials needed —
      # IRSA provides credentials automatically via the service account
```

**Key differences from `values-local.yaml`:**

| Field | Local (MinIO) | EKS (S3) |
|---|---|---|
| `config.storage.s3.endpoint` | `http://minio.qw.svc.cluster.local:9000` | *(omitted — uses AWS default)* |
| `config.storage.s3.flavor` | `minio` | *(omitted)* |
| `config.storage.s3.region` | `us-east-1` | `us-west-2` |
| `config.storage.s3.access_key_id` | `minioadmin` | *(omitted — IRSA)* |
| `config.storage.s3.secret_access_key` | `minioadmin123` | *(omitted — IRSA)* |
| `config.storage.s3.force_path_style_access` | `true` | *(omitted — virtual-hosted style)* |
| `serviceAccount.create` | *(default: true)* | `false` |
| `serviceAccount.name` | *(default)* | `quickwit-sa` |
| `environment.QW_METASTORE_URI` | `s3://quickwit-indexes/` | `s3://tokonoma-quickwit-indexes/` |
| `config.default_index_root_uri` | `s3://quickwit-indexes/` | `s3://tokonoma-quickwit-indexes/` |

### Deploy commands

```bash
# Helm repos (MinIO repo no longer needed)
helm repo add quickwit https://helm.quickwit.io
helm repo add open-telemetry https://open-telemetry.github.io/opentelemetry-helm-charts
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm repo add grafana https://grafana.github.io/helm-charts
helm repo update

# Namespace
kubectl create namespace qw --dry-run=client -o yaml | kubectl apply -f -

# Quickwit (S3 backend)
helm upgrade --install quickwit quickwit/quickwit \
  -n qw \
  -f 2o/platform/config/quickwit/values-eks.yaml

# OTel Collector
helm upgrade --install otel-collector open-telemetry/opentelemetry-collector \
  -n qw \
  -f 2o/platform/config/otel-collector/values.yaml

# Prometheus
helm upgrade --install prometheus prometheus-community/prometheus \
  -n qw \
  -f 2o/platform/config/monitoring/prometheus-values.yaml \
  --set server.persistentVolume.storageClass=gp2 \
  --set alertmanager.persistence.storageClass=gp2

# Grafana
helm upgrade --install grafana grafana/grafana \
  -n qw \
  -f 2o/platform/config/monitoring/grafana-values.yaml
```

## 10. Automation Script Plan

The existing `2o/platform/bin/setup` + `2o/platform/bin/deploy` handle local kind
clusters (with MinIO). An EKS automation script should:

1. **Prerequisites check** — verify awscli, eksctl, kubectl, helm, docker
2. **Cluster creation** — `eksctl create cluster` (idempotent: skip if exists)
3. **Post-cluster setup** — EBS CSI addon, IAM policy, kubeconfig
4. **S3 setup** — create S3 bucket, IAM policy, IRSA service account
5. **ECR setup** — create repos, build & push images (cross-arch)
6. **Platform deploy** — Quickwit with `values-eks.yaml` (S3 backend),
   OTel Collector, Prometheus (storageClass=gp2), Grafana
7. **Sock Shop deploy** — `./bin/build --eks && ./bin/deploy --eks`
8. **Load generators deploy** — `kubectl apply` with ECR image
9. **Quickwit indexes** — create via API (reuse existing helper)
10. **Verification** — wait for pods, print status and access instructions

### Key differences from local (kind) scripts

| Concern | Local (kind) | EKS |
|---|---|---|
| Cluster creation | `kind create cluster` | `eksctl create cluster` |
| StorageClass | `standard` | `gp2` |
| Object storage | MinIO (in-cluster) | S3 bucket |
| Storage credentials | Static keys in Quickwit values | IRSA (IAM role via service account) |
| Quickwit values | `values-local.yaml` | `values-eks.yaml` |
| Image loading | `kind load docker-image` | ECR push + pull |
| EBS volumes | N/A | Needs CSI driver addon + IAM policy |
| Cross-arch builds | N/A | `--platform linux/amd64` required |
| Metrics server | Manual install + patch | Built-in (or addon) |
